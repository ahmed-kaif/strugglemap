# StruggleMap — AI Math Animation Pipeline 🚀

This is the backend rendering engine for **StruggleMap**, built to dynamically generate localized, Manim-based mathematical animations using multi-agent LLM workflows. 

## 🛠 Tech Stack
* **Framework:** FastAPI (Python 3.11)
* **AI Orchestration:** LangChain & Vertex AI (Gemini 2.5 Flash & Pro)
* **Rendering Engine:** Manim Community Edition
* **Infrastructure:** Docker (Debian Slim with FFmpeg & LaTeX)

---

## 🛑 Prerequisites
Before running this locally, you must have the following installed on your machine:
1. [Docker](https://docs.docker.com/get-docker/)
2. [Google Cloud CLI (`gcloud`)](https://cloud.google.com/sdk/docs/install)
3. A Google Cloud Project with the **Vertex AI API enabled** and active billing.

---

## ⚡️ Quickstart: Local Development

### Step 1: Authenticate with Google Cloud
This backend uses Application Default Credentials (ADC) to securely talk to Vertex AI without hardcoded API keys.

Run this in your terminal to log in:
```bash
gcloud auth application-default login
```
Set your active project quota (replace `YOUR_PROJECT_ID` with your actual ID):
```bash
gcloud auth application-default set-quota-project YOUR_PROJECT_ID
```

### Step 2: Set Your Environment Variable
Store your project ID in your local terminal session:
```bash
# On Linux/macOS
export GCP_PROJECT_ID="YOUR_PROJECT_ID"

# On Windows (PowerShell)
$env:GCP_PROJECT_ID="YOUR_PROJECT_ID"
```

### Step 3: Build the Docker Image
Because Manim requires heavy system dependencies like FFmpeg and a full LaTeX distribution (dvisvgm), we run everything inside a container.

```bash
docker build -t strugglemap-backend .
```
*(Note: The first build will take a few minutes to download the LaTeX packages. Subsequent builds are cached and instant).*

### Step 4: Run the Container
Spin up the FastAPI server, exposing port `8000` and mounting your local Google Cloud credentials so the container can authenticate:

**Linux/macOS:**
```bash
docker run -d \
  -p 8000:8000 \
  -v ~/.config/gcloud:/root/.config/gcloud \
  -e GOOGLE_APPLICATION_CREDENTIALS=/root/.config/gcloud/application_default_credentials.json \
  -e GCP_PROJECT_ID=$GCP_PROJECT_ID \
  --name strugglemap-api \
  strugglemap-backend
```

**Windows (PowerShell):**
```powershell
docker run -d `
  -p 8000:8000 `
  -v ${env:USERPROFILE}\.config\gcloud:/root/.config/gcloud `
  -e GOOGLE_APPLICATION_CREDENTIALS=/root/.config/gcloud/application_default_credentials.json `
  -e GCP_PROJECT_ID=$env:GCP_PROJECT_ID `
  --name strugglemap-api `
  strugglemap-backend
```

To view the live server logs:
```bash
docker logs -f strugglemap-api
```

## 🧪 Testing the API

Once the container is running, access the interactive API documentation automatically generated by FastAPI:

📍 **Go to:** [http://localhost:8000/docs](http://localhost:8000/docs)

1. Open the `POST /generate-lesson` endpoint.
2. Click **"Try it out"**.
3. Enter a mathematical topic (e.g., `Domain of a Parabola`).
4. Click **Execute**.

*Note: Generating a response will take 15–30 seconds as the AI agents write the script, generate the Python code, and the Manim subprocess renders the `mp4` frames.*

The response will return a structured JSON containing the concept breakdown and a direct `video_url` you can click to watch the generated animation.
